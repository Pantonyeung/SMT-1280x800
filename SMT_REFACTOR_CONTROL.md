# SMT 1280×800 視覺鎖定式融入重構

## 最高原則

1. 現有視覺效果視為驗收基準，不作大面積清理。
2. 未完成同畫面對比前，不刪除任何補丁、外掛或覆蓋。
3. 每次只轉移一層責任，確認公網畫面一致後才進入下一層。
4. Sunmi T2S 1280×800 橫屏為唯一主要驗收尺寸。
5. 功能、觸控、捲動、彈窗及鍵盤行為必須與視覺一同驗收。

## 分層次序

### 第0層：原始基準
- 完整保存上載版本。
- 不改視覺、不改功能、不移除補丁。
- 建立可回退提交。

### 第1層：App Shell
範圍：
- 1280×800 主畫布
- 頂部狀態列
- 主內容可用高度
- 底部五項導航
- 全局 overflow 與安全區

公網驗收：
- 首頁、點單、訂單、堂食、售罄、更多頁切換時，頂欄與底欄不跳動。
- 畫面無垂直或水平外溢。
- 底部導航不遮住內容。
- 所有頁面主內容高度一致。

### 第2層：點單三欄版面
範圍：
- 分類欄
- 商品區
- 購物車
- 25/75、30/70、32/68 比例設定
- 各區獨立捲動

公網驗收：
- 分類、商品、購物車比例與現有接受版本一致。
- 切換比例後真正生效。
- 商品卡不被裁切。
- 購物車底部操作不被遮擋。

### 第3層：商品卡與分類
範圍：
- 純文字卡、小圖卡、大圖卡
- 價格、副標題、售罄、選中狀態
- 分類列與搜尋入口

公網驗收：
- 同屏商品數量與目前基準相若。
- 文字不過細、不重疊、不截斷主要資料。
- 點擊範圍正確。

### 第4層：購物車與修改流程
範圍：
- 加減數量
- 修改卡
- 套餐配對
- 快捷飲品
- 結帳入口

公網驗收：
- 加入商品後版面不跳位。
- 修改卡位置、寬度及高度符合鎖定要求。
- 背景不可誤觸。
- 底部按鈕固定但不遮內容。

### 第5層：共用彈窗系統
範圍：
- 遮罩
- 返回與確認
- 最大高度與內部捲動
- 鍵盤彈出處理

公網驗收：
- 彈窗不超過購物車高度。
- 背景完全鎖定。
- 長內容可於彈窗內捲動。
- 鍵盤出現後確認按鈕仍可操作。

### 第6層：其他營運頁
範圍：
- 訂單
- 堂食
- 售罄
- 更多
- 報表與設定

公網驗收：
- 每次只驗收一個頁面模組。
- 不以其他頁面順便改動作為同批工作。

## 每層完成條件

1. 已推上 main。
2. 有獨立提交編號。
3. 有清楚列出本次改動範圍。
4. 有清楚列出公網驗收頁面、操作及不應改變項目。
5. 使用者確認後才進入下一層。
6. 如驗收失敗，只回退或修正該層，不擴散修改其他層。
